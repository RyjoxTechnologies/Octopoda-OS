The Octopoda
Integration Guide

What this guide teaches: how to add persistent memory to an AI agent in any framework. What you'll have at the end of each section: an agent that remembers something, closes, reopens, and still remembers it. Time per section: 2 minutes.

Before you start (do this once)

Install Python 3.9 or newer. Check: 
python --version
If this shows 3.8 or lower, stop. Upgrade Python first. Nothing below will work otherwise.
Install Octopoda: 
pip install octopoda
Should see: Successfully installed octopoda-3.1.3. If not, see Troubleshooting A at the bottom.
Get your API key. Go to https://octopodas.com/signup, verify your email, copy the key from the dashboard. Keep it somewhere.
Set the key as an environment variable. Every example below assumes this is set. 
Mac/Linux:        export OCTOPODA_API_KEY=sk-octopoda-YOUR_ACTUAL_KEY
Windows PS:       $env:OCTOPODA_API_KEY="sk-octopoda-YOUR_ACTUAL_KEY"
Windows CMD:      set OCTOPODA_API_KEY=sk-octopoda-YOUR_ACTUAL_KEY
Replace YOUR_ACTUAL_KEY with your real key. It starts with sk-octopoda-.
Verify the key works. Run this: 
import os
from octopoda import AgentRuntime
agent = AgentRuntime("setup_check")
print(agent.remember("hello", "world"))
You should see MemoryResult(... success=True ...). If you see success=False or an auth error, see Troubleshooting B.
Section 1

The basics (no framework)

Use this if you're writing raw Python without LangChain, CrewAI, etc.

import os
os.environ['OCTOPODA_API_KEY'] = 'sk-octopoda-YOUR_KEY'  # or set via export

from octopoda import AgentRuntime

# Create an agent (agent_id is any string you want)
agent = AgentRuntime("my_first_agent")

# Write a memory
agent.remember("user:name", "Mike")
agent.remember("user:project", "AI startup")

# Read it back
print(agent.recall("user:name"))
# RecallResult(value='Mike', key='user:name', latency_us=61187.1, found=True)

Verify it persists across sessions. Close Python, reopen, run:

from octopoda import AgentRuntime
agent = AgentRuntime("my_first_agent")
print(agent.recall("user:name").value)   # → "Mike"
If you see "Mike", you're done. You now have persistent memory.
If it shows None or found=False: go to Troubleshooting C.
Section 2

LangChain

What you'll build: a LangChain conversation that remembers what the user said across Python restarts.

Install:

pip install octopoda langchain-core

The code:

import os
os.environ['OCTOPODA_API_KEY'] = 'sk-octopoda-YOUR_KEY'

from octopoda import LangChainMemory

# Create memory. agent_id = which user or session.
# session_id = optional sub-namespace.
memory = LangChainMemory(agent_id="my_bot", session_id="user_mike")

# Normally your LangChain ConversationChain calls these for you.
# Shown explicitly here so you can see what's happening.
memory.save_context(
    {"input": "Hi, my name is Mike."},
    {"output": "Nice to meet you, Mike!"},
)
memory.save_context(
    {"input": "I work on AI agents."},
    {"output": "Cool — what kind of agents?"},
)

# Next turn — LangChain loads the history automatically:
history = memory.load_memory_variables({})
print(history)
# → {'history': "Human: Hi, my name is Mike.\nAI: Nice to meet you, Mike!\n
#               Human: I work on AI agents.\nAI: Cool — what kind of agents?"}

Proof it persists. Close Python. Open again. Run:

from octopoda import LangChainMemory
memory = LangChainMemory(agent_id="my_bot", session_id="user_mike")
print(memory.load_memory_variables({}))

You see the full history from the previous session.

Using it with a real LangChain ConversationChain:

from langchain.chains import ConversationChain
from langchain_openai import ChatOpenAI  # or any LangChain LLM

llm = ChatOpenAI(model="gpt-4o-mini")
chain = ConversationChain(llm=llm, memory=memory)
chain.predict(input="What's my name?")  # → "Your name is Mike."

Troubleshooting LangChain: see Section TS-LC at the bottom.

Section 3

CrewAI

What you'll build: a multi-agent crew where results from one task are retrievable by a different agent in the next run.

Install:

pip install octopoda crewai

The code:

import os
os.environ['OCTOPODA_API_KEY'] = 'sk-octopoda-YOUR_KEY'

from octopoda import CrewAIMemory

# crew_id identifies the crew (one project = one crew_id)
memory = CrewAIMemory(crew_id="research_crew")

# Store a task result. Three args, in order: task_name, result, agent_role.
memory.store_task_result(
    task_name="market_scan",
    result="Found 3 emerging competitors in vector DB space",
    agent_role="researcher",
)

# Store a longer-term finding. Different signature: agent_role, key, finding.
memory.store_finding(
    agent_role="analyst",
    key="top_competitor",
    finding="Mem0 — most keyword-matched in HN discussions",
)

# Retrieve later (from a fresh Python session, even)
findings = memory.get_all_findings()
print(findings)
# → [{'key': 'top_competitor', 'data': {...'value': 'Mem0 — most keyword...'}}]
Common trap: the two methods have different arg orders. Don't mix them up:
• store_task_result(task_name, result, agent_role) — agent_role is LAST
• store_finding(agent_role, key, finding) — agent_role is FIRST
Section 4

AutoGen

What you'll build: a group chat where messages between multiple agents are stored and replayable.

Install:

pip install octopoda pyautogen

The code:

import os
os.environ['OCTOPODA_API_KEY'] = 'sk-octopoda-YOUR_KEY'

from octopoda import AutoGenMemory

# Use group_id (NOT agent_id). One group chat = one group_id.
memory = AutoGenMemory(group_id="research_team")

# Store messages — three positional args: sender, recipient, content
memory.store_message("user_proxy", "researcher", "Analyze the vector DB market")
memory.store_message("researcher", "user_proxy", "Market size is $2.1B, growing 38% YoY")
memory.store_message("user_proxy", "analyst", "Who are the top 3?")
memory.store_message("analyst", "user_proxy", "Pinecone, Weaviate, Qdrant")

# Retrieve history
history = memory.get_conversation_history()
for msg in history:
    print(f"{msg['sender']} → {msg['recipient']}: {msg['content']}")
The key gotcha: it's group_id, NOT agent_id. If you pass agent_id="x" you'll get TypeError: __init__() got an unexpected keyword argument 'agent_id'.
Section 5

OpenAI Agents SDK

What you'll build: an OpenAI assistant whose thread state persists across server restarts.

Install:

pip install octopoda openai

The code:

import os
os.environ['OCTOPODA_API_KEY'] = 'sk-octopoda-YOUR_KEY'

from octopoda import OpenAIAgentsMemory

# No arguments — the class takes none.
memory = OpenAIAgentsMemory()

# Save conversation thread state mid-flight
memory.store_thread_state(
    thread_id="thread_12345",
    state={
        "messages": [
            {"role": "user", "content": "Help me plan a trip to Tokyo"},
            {"role": "assistant", "content": "Sure. How many days?"},
        ],
        "step": 1,
    },
)

# Save a completed run's outcome
memory.store_run_result(
    run_id="run_67890",
    result="Completed itinerary: 5 days, 4 neighborhoods, $3200 budget",
)

# Next session — restore the thread
restored = memory.restore_thread("thread_12345")
print(restored["state"]["messages"])
The key gotcha: OpenAIAgentsMemory() takes NO arguments. Don't pass agent_id or anything. It manages agent identity internally through thread_id and run_id.
Section 6

MCP server (Claude Code, Cursor, Windsurf, Codex)

What you'll build: Claude Code (or Cursor/Codex) gets 28+ memory tools that persist across sessions.

Install the Octopoda MCP extras:

pip install "octopoda[mcp]"

For Claude Code, add to your ~/.claude/mcp.json:

{
  "mcpServers": {
    "octopoda": {
      "command": "python",
      "args": ["-m", "synrix_runtime.api.mcp_server"],
      "env": {
        "OCTOPODA_API_KEY": "sk-octopoda-YOUR_KEY"
      }
    }
  }
}

Want to use it WITHOUT a cloud account? Set the env var to any string:

"env": { "OCTOPODA_API_KEY": "local" }

This runs in local mode using SQLite at ~/.synrix/data/synrix.db.

Verify it works in your terminal first:

OCTOPODA_API_KEY="sk-octopoda-YOUR_KEY" python -m synrix_runtime.api.mcp_server

You should see no errors. It's waiting for JSON-RPC input. Press Ctrl+C to exit.

Now restart Claude Code. Ask: "Use octopoda_remember to store my name as Mike." Claude will use the tool. Ask in a new session: "Use octopoda_recall to get my name." Claude returns "Mike."

Section 7

OpenClaw (Node.js, different path)

OpenClaw is a Node.js agent, not a Python framework. Integration is via a downloadable skill file, not pip.

Requirements:

  • Node.js installed
  • OpenClaw installed: npm install -g openclaw
  • The Octopoda Memory skill file (download from octopodas.com/docs/integrations/openclaw)

Drop the skill file into your OpenClaw agent's skill directory. Set OCTOPODA_API_KEY in the environment. Done.

Troubleshooting

TS-A. pip install octopoda fails

If you see error: Microsoft Visual C++ 14.0 is required (Windows only):

  1. Go to visualstudio.microsoft.com/visual-cpp-build-tools/
  2. Install the Build Tools (pick "Desktop development with C++")
  3. Restart your terminal, retry pip install

If you see ERROR: Could not find a version that satisfies the requirement octopoda:

  • Check Python version: python --version. Must be 3.9 or newer.
  • Try pip install --upgrade pip first, then retry

If pip hangs forever with no output: run pip install -v octopoda. Usually a network issue or corporate firewall blocking PyPI.

TS-B. AgentRuntime created but writes fail with 401

You'll see either "Authentication required" or success=False.

  • Check your key is set: python -c "import os; print(os.environ.get('OCTOPODA_API_KEY'))"
  • If it shows your key but auth fails: it may have been rotated, or there's a whitespace character pasted in. Regenerate at /dashboard/settings.
  • If it shows None: the env var isn't being picked up. On Windows, setting it in CMD doesn't persist to a new terminal. Set it inside your script: os.environ['OCTOPODA_API_KEY'] = 'sk-octopoda-...'

TS-C. Memory not persisting between Python sessions

  • Use the SAME agent_id in both sessions. AgentRuntime("bot_a") and AgentRuntime("bot_b") are two different agents.
  • If using local mode: memory lives at ~/.synrix/data/synrix.db. Delete that file or switch user account → memory gone.
  • If cloud mode and recall returns found=False right after a write: wait 1–2 seconds and retry. Very rare read-after-write cache miss.

TS-LC (LangChain). save_context works but load_memory_variables returns empty

  • Use the same session_id across save_context and load_memory_variables.
  • Make sure save actually ran before load — not just in the script, but in time.
  • Don't wrap with LangChain's ConversationBufferMemory. LangChainMemory is a drop-in replacement, not additive.

TS-3 (CrewAI). store_finding gives TypeError

Wrong arg name. It is agent_role, key, finding — in that order. Not name, not content.

memory.store_finding(agent_role="researcher", key="x", finding="Y")

TS-4 (AutoGen). TypeError: unexpected keyword argument 'agent_id'

memory = AutoGenMemory(group_id="my_group")  # not agent_id=

TS-6 (MCP). MCP server exits immediately

If you see ERROR: OCTOPODA_API_KEY not set: you're on an old version. Upgrade:

pip install --upgrade "octopoda[mcp]"

Version 3.1.3+ supports local mode without a key.

If Claude Code says "MCP server not found": check the command path in ~/.claude/mcp.json. Use the absolute path if needed: /usr/local/bin/python3 or C:\Python313\python.exe.
